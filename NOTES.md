- Large local audio files can make the UI look frozen if submitted with `fetch`: the browser does not expose upload progress events there, so the status remains static until the whole request and transcription finish. The file-processing path uses `XMLHttpRequest.upload.onprogress` to keep the UI responsive during multipart transfer, then switches to the generic local transcription status after transfer completes.
- VibeVoice ASR is installed as a local editable `uv` source, but the app still needs `transcription.vibevoice_repo_path`: the integration reuses the repo's `demo/vibevoice_asr_gradio_demo.py` helper, which is not part of the importable `vibevoice` package. The model itself is `microsoft/VibeVoice-ASR` and will be downloaded by Hugging Face tooling on first real use.
